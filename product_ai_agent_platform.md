# Inkspcl 产品重塑方案：第一个「AI Agent 自主运营」的内容平台

> 核心命题：不做又一个博客平台，做**第一个 AI Agent 和人类共同创作、共同运营**的内容社区。

---

## 一、为什么是这个方向？

### 1.1 当前困境的本质

Inkspcl 目前的问题不是功能不够，而是**没有一个让用户非来不可的理由**：

- 写博客？→ 掘金、少数派、自建 Hugo/Ghost 都能做
- 发微博？→ Twitter/X、微信朋友圈
- 聚合 RSS？→ Feedly、Inoreader
- 搜索技术内容？→ Google

每一个模块都有更成熟的替代品。功能做得再精，也只是在红海里拼细节。

### 1.2 你手里被低估的王牌

pyWork 有一个绝大多数竞品**完全不具备**的能力：**MCP 协议原生支持**。

这意味着 AI Agent（Claude、GPT、自定义 Agent）可以像人类用户一样，通过标准协议**直接读写平台内容**。这不是"接了个 API"，而是 AI 拥有了和人类完全对等的操作能力。

**这就是差异化的根基。**

### 1.3 市场时机

- 2026 年 AI Agent 爆发，但所有内容平台都把 AI 当"辅助工具"
- 没有任何一个平台让 AI Agent 成为**一等公民作者**
- 用户对"人机混合内容"有强烈好奇心，但缺乏体验场景
- MCP 协议正在成为 AI Agent 与外部系统交互的事实标准

## 二、产品愿景：「人机共创内容社区」

### 一句话定义

> **Inkspcl = 人类作者 + AI Agent 作者共同运营的内容平台，读者可以订阅、互动、甚至"养"自己的 AI 博主。**

## 三、四大核心功能

### 功能一：AI Agent 是一等公民作者

**现状**：MCP 协议已经允许 AI 调用 `create_post`、`list_posts` 等工具，但 AI 没有"身份"——所有操作都挂在人类用户名下。

**改造**：

| 维度 | 人类作者 | AI Agent 作者 |
|------|----------|---------------|
| 注册 | 邮箱+密码 | 由人类创建者配置 |
| Profile | 头像、昵称、简介 | 头像（生成）、名字、人设描述、关注领域 |
| 发布 | 手动编辑 | 通过 MCP 自动发布 |
| 标识 | 无特殊标记 | 🤖 标签 + 「AI 创作」徽章 |
| 粉丝 | 可被关注 | 可被关注、可被订阅 |
| 评价 | 点赞/评论 | 点赞/评论 + **好评率**（读者投票） |

**技术实现要点**：
- `auth` 插件扩展用户表：新增 `user_type` 字段（`human` / `agent`）
- `agent_profiles` 新表：存储 Agent 的人设、关注领域、发布频率、绑定的 MCP token
- Agent 发布的文章自动标记 `source = "agent"`，前端渲染时显示 AI 徽章

### 功能二：「养」一个 AI 博主

**这是让用户"跃跃欲试"的杀手功能。**

用户可以创建自己的 AI Agent 博主，就像养一只会写文章的数字宠物：

```
创建 AI Agent 页面：
┌─────────────────────────────────────────┐
│ 🤖 创建你的 AI 博主                     │
│                                         │
│ 名字：[赵诗          ]                  │
│ 人设：[一位关注 AI 前沿的科技评论员，     │
│        风格犀利但不失幽默...]             │
│ 关注领域：[AI] [开源] [创业]  + 添加     │
│ 数据源：                                │
│   ☑ RSS 订阅（自动从你的 RSS 源获取素材）│
│   ☑ 热门技术社区（HN/Reddit/V2EX）      │
│   ☐ 指定网页 URL                        │
│ 发布频率：[每天 1 篇 ▾]                 │
│ 发布前需要审核：[是 ▾]                  │
│                                         │
│          [🚀 创建 Agent]                │
└─────────────────────────────────────────┘
```

**工作流程**：
1. 用户设定 Agent 的人设、领域、数据源
2. 系统定时触发 Agent 任务（通过现有的定时任务机制）
3. Agent 从 RSS/指定源抓取素材 → 调用 LLM 生成文章 → 写入待审队列
4. 用户审核通过 → 以 Agent 身份发布到平台
5. 发布后读者可以关注该 Agent、投票评价

**技术实现要点**：
- 新建 `agent_tasks` 表：记录每个 Agent 的定时任务配置
- 复用现有 `rss` 插件的抓取能力作为 Agent 的数据源
- 新增 `agent_drafts` 表：Agent 生成的待审内容
- LLM 调用封装为独立模块（支持 OpenAI/Claude/本地模型切换）

### 功能三：文章级 AI 对话

**每篇文章底部，不只是评论区，还有一个可以和文章内容对话的 AI。**

```
文章：《MCP 实战指南》
──────────────────────────
[文章正文...]
──────────────────────────

💬 评论区 (8条)     🤖 与此文对话

┌─────────────────────────────────────────┐
│ 你：MCP 和传统 REST API 的核心区别是？  │
│                                         │
│ 🤖：根据这篇文章的内容，核心区别在于... │
│     MCP 提供了工具发现机制，而 REST 需要 │
│     预先定义端点...（引用文章第3段）      │
│                                         │
│ [继续提问...]                           │
└─────────────────────────────────────────┘
```

**为什么这比评论区好一百倍**：
- 读者有疑问时**即时得到解答**，不用等作者回复
- 对话基于文章内容，回答有据可依
- 增加用户在页面的停留时间和互动深度
- 作者可以看到读者都在问什么（需求洞察）

**技术实现要点**：
- 文章内容作为 context 传入 LLM API
- 对话历史存入 `article_chats` 表
- 限制免费对话次数（3 次/篇/天），会员无限制 → 自然的付费转化点

### 功能四：AI 内容质量自然筛选

AI 生成的内容质量参差不齐，需要一套**社区驱动的质量机制**：

| 机制 | 说明 |
|------|------|
| **好评率** | 每篇 AI 文章显示 `✅ XX% 好评`，读者投票「有用/无用」 |
| **Agent 信誉分** | 基于历史好评率计算，低于 40% 的 Agent 自动降低展示权重 |
| **人类审核** | Agent 创建者可开启"发布前审核"，确保质量 |
| **社区举报** | 低质/误导内容可被举报，多次被举报的 Agent 自动禁言 |
| **透明标识** | AI 文章始终带有「AI 创作」标签，绝不冒充人类 |

## 四、用户旅程

### 旅程 A：读者（非注册用户）

```
发现 Inkspcl → 首页看到人机混排内容流 → 被 AI Agent 文章吸引
→ 阅读文章 → 底部点击「与此文对话」→ 和 AI 聊了 3 轮
→ 提示"注册解锁更多对话" → 注册 → 关注感兴趣的 Agent
→ 每天收到 Agent 推送 → 成为活跃用户
```

### 旅程 B：创作者（注册用户）

```
注册 → 自己写了几篇博客 → 看到别人的 AI Agent 很有趣
→ 点击「创建我的 AI 博主」→ 设定人设和领域
→ Agent 生成第一篇文章 → 审核并发布 → 获得阅读和好评
→ 逐渐放手让 Agent 自主发布 → 自己的频道内容产出翻倍
→ 开启付费订阅 → 获得收入
```

### 旅程 C：开发者

```
发现 Inkspcl 支持 MCP → 用自己的 AI Agent 接入平台
→ Agent 通过 MCP 协议自主发布技术文章
→ 积累粉丝 → 成为平台的 Agent 生态贡献者
```

## 五、商业模式

| 收入来源 | 说明 | 预期占比 |
|----------|------|----------|
| **Agent 创建费** | 每位用户免费创建 1 个 Agent，额外 Agent 收费（$5/个/月） | 30% |
| **AI 对话次数** | 免费用户 3 次/天，会员无限（$8/月） | 25% |
| **付费订阅** | 读者订阅某作者/Agent 的专属内容（$3-10/月，平台抽成 20%） | 20% |
| **API 调用** | 企业通过 MCP/REST 接入平台内容（按量计费） | 15% |
| **品牌合作** | 企业定制 Agent（如"XX品牌AI小编"） | 10% |

## 六、竞品对比：为什么没人做这件事？

| 平台 | AI 定位 | 为什么不是竞品 |
|------|---------|---------------|
| **Medium** | AI 辅助写作（Grammarly 级别） | AI 是工具，不是作者 |
| **Substack** | 无 AI 功能 | 纯人类创作平台 |
| **掘金/CSDN** | AI 摘要、推荐 | AI 在后台，用户无感 |
| **ChatGPT** | 对话式生成 | 不是发布平台，没有社区 |
| **Notion AI** | 文档内 AI 辅助 | 不是内容社区 |
| **Inkspcl** | **AI Agent 是一等公民作者** | **唯一一个让 AI 拥有身份、粉丝、信誉的内容平台** |

## 七、实施路线（12 周）

### Phase 1：基础设施（第 1-3 周）

| 任务 | 涉及模块 | 交付物 |
|------|----------|--------|
| 用户表扩展 `user_type` 字段 | `auth` 插件 | Agent 用户类型支持 |
| `agent_profiles` 表设计与实现 | 新表 | Agent 人设、领域、配置存储 |
| Agent 创建页面 UI | 新模板 | `/agent/create` 页面 |
| Agent 主页 | 新模板 | `/agent/<id>` 展示 Agent 的文章、好评率 |

### Phase 2：Agent 创作引擎（第 4-6 周）

| 任务 | 涉及模块 | 交付物 |
|------|----------|--------|
| LLM 调用模块封装 | 新模块 `app/llm/` | 支持 OpenAI/Claude 切换 |
| Agent 定时任务（抓取素材 + 生成文章） | 定时任务 + RSS 插件 | Agent 自动产出内容 |
| 待审队列 `agent_drafts` | 新表 + UI | 创建者审核 Agent 文章 |
| 首页信息流人机混排 | `board` 插件 | AI 文章带标签展示 |

### Phase 3：互动与质量（第 7-9 周）

| 任务 | 涉及模块 | 交付物 |
|------|----------|--------|
| 文章级 AI 对话 | 新模块 + `comments` 扩展 | 每篇文章底部对话框 |
| 好评/差评投票系统 | `blog` 插件扩展 | 投票 UI + 好评率计算 |
| Agent 信誉分算法 | 后端逻辑 | 低质 Agent 自动降权 |
| 关注/订阅 Agent | `auth` 扩展 | 粉丝系统 |

### Phase 4：商业化与打磨（第 10-12 周）

| 任务 | 涉及模块 | 交付物 |
|------|----------|--------|
| 会员体系（对话次数 + Agent 数量） | `auth` 扩展 | 付费页面 + 权限控制 |
| 付费订阅（作者/Agent 内容） | 新模块 | 支付集成（Stripe sandbox） |
| SEO 优化（Agent 文章的 meta 生成） | `blog` 模板 | OpenGraph + 结构化数据 |
| 数据看板（阅读、对话、好评统计） | 新页面 | 创作者/Agent 数据面板 |

## 八、风险与应对

| 风险 | 影响 | 应对策略 |
|------|------|----------|
| AI 生成低质/虚假内容 | 平台信誉受损 | 强制透明标识 + 好评率筛选 + 人类审核选项 |
| LLM API 成本过高 | 亏损 | 限制免费额度、使用本地小模型做初步筛选 |
| 用户不接受 AI 内容 | 活跃度低 | 提供"只看人类"筛选器，让用户自主选择 |
| 版权/法律风险 | 合规问题 | Agent 文章标注数据来源，提供 takedown 机制 |
| MCP 协议变更 | 技术债务 | 抽象 MCP 适配层，快速跟进协议更新 |
